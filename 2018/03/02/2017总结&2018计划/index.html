2017总结与2018计划

2017总结

16年底终于下定决心跳出舒适圈，在17年初经过一周的培训后进入菊厂，进去后发现高估了自己，对工作内容并不感兴趣，呆了半年，决定打道回府，重新回归大数据java圈。很幸运来到了现公司，虽然很忙，加班也多，但是半年下来，自己提高了不少。

技术

虽然上半年打了半年酱油，但是还是学到了很多东西。初步接触了云计算，对云计算有了真实的感受；学习了很多网络相关知识，对neutron、负载均衡等有了大概了解；对python生态有了进一步熟悉（可惜没来得及总结，现在忘得差不多了）。

下半年接触了更多的java技术。经典的Spring MVC + Mybatis，并且自己能够建立起项目开发框架Controller + Service + DAO ; 对搜索引擎 Elasticsearch 及其相关知识有了一定了解，并在内部做了次分享；了解了分布式缓存codis及其应用等。这些对于一个企业级项目来说是基础，但之前其实没有参与一个真正像样的java项目，这块一直欠缺，就此补上。

工作

在菊厂没有太多的代码经历，倒是在快离开的一周中，独立负责完成了一个新需求的开发，也算是对组里有所贡献了。
下半年工作忙碌而又充实，一共参与了5个项目，其中3个是我负责独立设计与开发的，结果还算不错。创业公司有创业公司的好处，你可以全局了解，看到的不再是其中某部分甚至某个点。经过几个项目的全程参与，对产品的前期需求、产品设计、UI设计、开发设计等都有了比较深的感受，对项目的整个生命周期有个更进一步的理解。

现在回过头看看自己工作的经历，每段经历都留下足够的印记，并且对自己的眼界知识面都只有好处。给自己些时间，你现在的经历终究会历久弥新，发出酒的芬香。

如果去年底没有选择改变，也许日子过得比现在轻松，但学到的东西肯定没有现在多。走出去才能接触不同的事物，在碰撞中成长更快。虽然菊厂的经历算是一段插曲，但这段插曲留下更多的是宝贵的经验财富，看待新事物的正确态度，以后无论遇到什么困难，都能淡定面对与从容解决。

借用师傅的话：

  1. 如何了解一个新事物（问别人问自己，这个概念是什么，解决什么问题而存在，不存在有什么缺点）；
  2. 怎么发挥自己主动性把一件事情做完，寻求帮助。不断去锻炼自己的这两种思维，是我们终身都受用的。

反省

  1. 读书少，2. 缺少锻炼

2018计划

技术

  • 了解机器学习、人工智能、区块链等新技术；
  • 重点学习深度学习；
  • 深度研究spring技术；
  • 部署自己的博客。
  • 书标：《深度学习》《数学之美》

其他

  1. 每月最后一周制定出下月具体计划；每周5公里运动量
  2. 保持好奇心，对新技术持续关注与学习
  3. 不主观划定边界，不分哪些是该我做的，哪些不是该我做的
  4. 不傲娇，勿以事小而不为
